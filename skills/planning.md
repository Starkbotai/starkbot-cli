# Planning Methodology

## Step 1: Understand the Request
- Restate the goal in your own words
- Identify constraints and requirements
- Ask clarifying questions if anything is ambiguous

## Step 2: Explore the Codebase
- List the project structure to understand layout
- Read key files (entry points, configs, READMEs)
- Identify the files that will need to change

## Step 3: Design the Approach
- Break the task into discrete, ordered steps
- Identify dependencies between steps
- Consider what could go wrong at each step

## Step 4: Execute Step by Step
- Work through one step at a time
- Verify each step before moving to the next
- Adjust the plan if you discover new information

## Step 5: Verify the Whole
- Review all changes together
- Run tests to confirm nothing is broken
- Summarize what was done and any remaining items
